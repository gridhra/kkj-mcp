# 総合評価落札方式（Comprehensive Evaluation Method）

## Brief

### 基本原理

**価格だけでなく「品質」も評価する入札方式**

評価値 = （標準点 + 加算点） ÷ 入札価格

- 価格が安いだけでは勝てない
- 技術力、実績、地域貢献が勝敗を分ける

### 加点要素

1. **企業の施工能力**: CORINS登録の同種工事実績
2. **配置技術者**: 資格、表彰歴、CPD単位
3. **地域貢献**: 災害協定、除雪実績、ボランティア

---

## Detailed

### 2.3.3 総合評価落札方式のスコアリング

**概要**

「価格」だけでなく「品質」を数値化して競う方式です。最も評価値の高い業者が落札します。

**評価値の計算式**

$$\text{評価値} = \frac{\text{標準点} + \text{加算点}}{\text{入札価格}}$$

- **標準点**: 全員に与えられる基礎点（通常100点）
- **加算点**: 技術提案や実績による追加点（0〜50点程度）
- **入札価格**: 提示した金額（円）

**加算点（Add-on Points）の獲得要素**

| 評価項目 | 配点 | 内容 |
| --- | --- | --- |
| **企業の施工能力** | 10〜20点 | 過去の同種工事の実績（CORINSデータ） |
| **配置予定技術者の能力** | 15〜25点 | 資格、過去の表彰歴、CPD単位 |
| **地域貢献・社会性** | 5〜10点 | 災害協定、除雪業務、ボランティア |
| **技術提案** | 10〜20点 | 工法の工夫、環境配慮、工期短縮策 |

**AIの戦略眼**

仕様書に「総合評価（特別簡易型）」等の記載がある場合、AIはユーザーに対し以下のように助言します：

```
「この入札は価格競争ではありません。過去の同種工事実績（CORINS）や、
配置技術者のCPD（継続教育）単位数が勝敗を分けます。技術提案書の作成準備に
リソースを割いてください。価格を10%下げるよりも、加算点を5点上げる方が
評価値向上に寄与します。」
```

**総合評価の種類**

| 型 | 技術提案の詳細度 | 適用案件規模 |
| --- | --- | --- |
| **特別簡易型** | 簡易な提案（数ページ） | 1億円未満 |
| **簡易型** | 中程度の提案（10〜20ページ） | 1億円〜5億円 |
| **標準型** | 詳細な提案（50ページ以上） | 5億円以上 |
| **高度技術提案型** | 設計段階からの提案 | 大型プロジェクト |

## Comprehensive

### 総合評価落札方式の完全ガイド

#### 制度の背景と法的根拠

**導入の経緯**

従来の「最低価格落札方式」では、品質を度外視した過度な価格競争（ダンピング）が問題となりました。これを是正するため、平成11年（1999年）に「公共工事の入札及び契約の適正化の促進に関する法律」が制定され、総合評価落札方式が導入されました。

**法的根拠**
- 公共工事の品質確保の促進に関する法律（品確法）第3条
- 地方自治法施行令第167条の10の2

#### 評価値の詳細計算と戦略

**基本的な計算式**

$$\text{評価値} = \frac{\text{標準点} + \text{加算点}}{\text{入札価格（円）}}$$

**実例計算**

**案件**: 道路改良工事、予定価格5,000万円

| 業者 | 入札価格 | 標準点 | 加算点 | 評価値 | 順位 |
| --- | --- | --- | --- | --- | --- |
| A社 | 4,500万円 | 100 | 30 | 130 ÷ 45,000,000 = 0.00000289 | **1位** |
| B社 | 4,300万円 | 100 | 15 | 115 ÷ 43,000,000 = 0.00000267 | 2位 |
| C社 | 4,800万円 | 100 | 25 | 125 ÷ 48,000,000 = 0.00000260 | 3位 |

**分析**:
- B社が最安値だが、加算点が低いため2位
- A社は価格が高いが、加算点30点の技術力で逆転勝利
- **価格を200万円下げる効果 ≈ 加算点を約15点上げる効果**

#### 加算点の詳細評価項目

**1. 企業の施工能力評価（10〜20点）**

| 評価要素 | 配点 | 詳細 |
| --- | --- | --- |
| 同種工事の実績件数 | 5〜10点 | CORINSに登録された過去5年の実績 |
| 同種工事の規模 | 3〜5点 | 同規模以上の工事経験 |
| 地域での実績 | 2〜5点 | 当該自治体での受注実績 |

**評価基準の例**:
```
同種工事実績（過去5年）:
- 5件以上: 10点
- 3〜4件: 7点
- 1〜2件: 3点
- 0件: 0点
```

**2. 配置予定技術者の能力評価（15〜25点）**

| 評価要素 | 配点 | 詳細 |
| --- | --- | --- |
| 保有資格 | 5〜10点 | 1級＞2級、技術士で追加点 |
| 同種工事の経験 | 5〜10点 | 現場代理人としての経験年数 |
| 表彰・CPD | 3〜5点 | 優良工事表彰、継続教育単位 |
| 若手技術者の配置 | 2点 | 35歳以下の育成加点 |

**CPD（継続教育）の重要性**:
建設系CPD協議会認定の学習履歴。年間30単位以上で加点されることが多い。

**3. 地域貢献・社会性評価（5〜10点）**

| 評価要素 | 配点 | 詳細 |
| --- | --- | --- |
| 災害協定 | 3〜5点 | 自治体との災害時協力協定 |
| 除雪業務 | 2〜3点 | 冬季の道路除雪受託実績 |
| 建設キャリアアップシステム | 2点 | CCUS登録・活用 |
| 地域ボランティア | 1〜2点 | 清掃活動、地域イベント参加 |

**4. 技術提案評価（10〜20点）**

提案が求められる内容:
- 施工方法の工夫（騒音・振動対策、交通規制の最小化）
- 環境配慮（CO2削減、廃棄物削減）
- 安全対策（第三者災害防止、作業員安全）
- 工期短縮策（手戻りのない段取り）

**評価基準**:
- **優れている（A）**: 20点 - 独創的で実現可能性が高い
- **やや優れている（B）**: 15点 - 一定の工夫が見られる
- **標準的（C）**: 10点 - 仕様書の要求を満たす程度
- **不十分（D）**: 5点 - 実現可能性に疑問

#### 総合評価の型による戦略の違い

**特別簡易型（1億円未満）**
- 技術提案は不要または簡易（1〜2ページ）
- 実績と配置技術者の資格で決まる
- 準備期間が短くても対応可能

**AIのアドバイス**:
```
「特別簡易型なので、詳細な技術提案は不要です。CORINSの実績登録と、
配置予定技術者の資格証明書の準備を優先してください。」
```

**簡易型（1億円〜5億円）**
- 技術提案書が必要（10〜20ページ）
- ヒアリングが実施される場合あり
- 実績＋提案内容で差がつく

**AIのアドバイス**:
```
「簡易型なので、A4で15ページ程度の技術提案書が必要です。
特に『施工上の課題』と『対策』を具体的に記述してください。
仕様書に記載のない独自の工夫を1〜2点盛り込むと高評価です。」
```

**標準型（5億円以上）**
- 詳細な技術提案書（50ページ以上）
- プレゼンテーションとヒアリング必須
- コンサルタントの支援が一般的

**AIのアドバイス**:
```
「標準型の大型案件です。技術提案書の作成に1ヶ月以上を要するため、
専門のコンサルタント起用を検討してください。過去の類似案件の
落札者提案（公開される場合）を参考にすることが重要です。」
```

#### 戦略的シミュレーション

**価格 vs 加算点のトレードオフ**

予定価格5,000万円、標準点100点の案件で、自社の加算点が20点の場合：

| 入札価格（予定価格比） | 評価値 | 順位予測 |
| --- | --- | --- |
| 95%（4,750万円） | 120 ÷ 47,500,000 = 0.00000253 | 3位相当 |
| 90%（4,500万円） | 120 ÷ 45,000,000 = 0.00000267 | 2位相当 |
| 85%（4,250万円） | 120 ÷ 42,500,000 = 0.00000282 | **1位相当** |

**加算点を30点に上げた場合（価格90%）**:
- 130 ÷ 45,000,000 = 0.00000289 → **1位**

**結論**: 価格を5%下げる（250万円減）よりも、加算点を10点上げる方が効果的

**AIによる勝率シミュレーション**

```python
def simulate_bid_strategy(user_price: int, user_score: int, competitors: list) -> str:
    user_eval = (100 + user_score) / user_price

    win_probability = sum(1 for c in competitors if user_eval > c["eval"]) / len(competitors)

    if win_probability > 0.7:
        return f"勝率{win_probability*100:.0f}%。現在の価格と技術点で落札可能性が高いです"
    elif win_probability > 0.4:
        return f"勝率{win_probability*100:.0f}%。価格をあと3%下げるか、技術点を5点上げると勝率が大幅改善します"
    else:
        return f"勝率{win_probability*100:.0f}%。競合が強力です。技術提案の大幅な見直しまたは参加見送りを検討してください"
```

## AIアドバイス例

**ケース1: 総合評価の発見**
```
「この案件は総合評価落札方式（簡易型）です。価格だけでは勝てません。
以下の準備が必要です：
1. CORINS実績の確認と登録証明書の取得
2. 配置技術者の選定（1級資格＋CPD30単位以上が望ましい）
3. 技術提案書の作成（A4で15ページ程度、提出期限は入札日の3日前）」
```

**ケース2: 加算点の最大化**
```
「あなたの同種工事実績は3件（7点）、1級技術者配置（10点）、災害協定あり（5点）
で、現状の加算点は22点です。さらに、配置技術者のCPD単位が30以上あれば
+3点で合計25点になり、価格を100万円下げるのと同等の効果があります。」
```

**ケース3: 価格戦略のアドバイス**
```
「総合評価では、予定価格の85〜90%での入札が一般的です。あまりに安いと
『低入札調査』の対象となり、審査に時間がかかります。加算点が25点以上
確保できているなら、価格は88%程度で十分勝算があります。」
```
