# 資格要件とランク付け（Qualifications and Rankings）

## Brief

### 参加資格の3つの軸

1. **経審ランク（A/B/C/D）** - 企業規模と技術力の評価（P点で決定）
2. **地域要件** - 市内業者/準市内業者/県内業者/県外可
3. **配置技術者** - 1級・2級施工管理技士等の資格保有

### 重要ポイント

- **P点**: 経営事項審査の総合評定値。ランク区分の基準
- **地域要件**: 小規模案件ほど地元企業に限定される傾向
- **CORINS**: 過去の工事実績データベース。総合評価で重要

## Detailed

### 2.3 公告文書の解読：参加資格と「足切り」の論理

`get_notice_details`で取得した詳細は、単なるテキストではなく「参加フィルター」の集合体です。AIは以下の3つのベクトルで適合性を判定する知識を持つ必要があります。

### 2.3.1 経営事項審査（経審）とランク付け（Ranking）

**概要**

日本の公共工事は、企業の規模や能力に応じてランク（A, B, C, D）分けされています。これは**経営事項審査（経審）**の総合評定値（**P点**）に基づきます。

**P点の計算式**

$$P = 0.25 X_1 + 0.15 X_2 + 0.20 Y + 0.25 Z + 0.15 W$$

各要素の意味：
- **X1（完成工事高）**: 企業の基礎体力。年間売上
- **X2（自己資本額・利益額）**: 財務の健全性
- **Y（経営状況）**: 負債比率、純支払利息比率など
- **Z（技術力）**: 1級土木施工管理技士などの有資格者数
- **W（社会性等）**: 防災協定、建設キャリアアップシステム（CCUS）の導入、若年者育成など

**地域による偏差**

東京都のAランクはP点900点以上が必要ですが、地方の小規模自治体では700点でAランクとなることもあります。

**AIの挙動**

ユーザーのP点が「850点」であると仮定した場合：
- 東京都のAランク案件 → 「ランク不足の可能性が高い」と警告
- Cランク案件 → 「ランク過多により参加できない可能性があります（不調対策を除く）」と案内

### 2.3.2 地域要件（Geographic Restrictions）

**区分の厳格性**

「市内業者」「準市内業者（支店あり）」「県内業者」の区分は厳格です。

**ドメイン知識**

地方自治体は地元企業育成のため、小規模案件（例：2,000万円未満）を徹底して「市内本店」に限定します。一方で、高度な技術を要する「シールドトンネル工事」や「橋梁補修」などは、地域要件を緩和して県外の大手ゼネコンを誘致します。

**AIの検索ヒント**

ユーザーが特定地域外の企業である場合、AIは検索キーワードに「ＷＴＯ」や「公募型」を含めるか、あるいは特殊な工種（法面処理、PC橋上部工など）に絞るよう提案すべきです。一般的な「道路改良」や「舗装」は、地域外企業が入れる余地はほぼありません。

## Comprehensive

### 経営事項審査（経審）の完全ガイド

#### 経審制度の目的と法的根拠

**法的根拠**: 建設業法第27条の23〜27

公共工事を受注するには、経営事項審査（経審）を受けることが義務付けられています。これは企業の「健康診断」のようなもので、以下を数値化します：

1. 経営規模（売上高、自己資本）
2. 経営状況（財務指標）
3. 技術力（技術者数、工事実績）
4. 社会性（法令遵守、環境対策、働き方改革等）

#### P点の詳細計算式と戦略

**完全な計算式**

$$P = 0.25 X_1 + 0.15 X_2 + 0.20 Y + 0.25 Z + 0.15 W$$

各要素の詳細：

**X1: 経営規模（完成工事高）**
- 直近2年または3年の平均年間完成工事高
- 業種別に評点化（土木、建築、電気、管など）
- 最大値: 約2,300点（年間売上1,000億円以上の企業）

**X2: 経営規模（自己資本額・利益額）**
- 自己資本額と平均利益額の合計
- 財務の安定性を評価
- 最大値: 約2,280点

**Y: 経営状況**
- 8つの財務指標を分析（純支払利息比率、負債回転期間など）
- 倒産リスクの低さを評価
- 範囲: 0〜1,595点（マイナスもあり得る）

**Z: 技術力**
- 技術職員数（1級施工管理技士、技術士等）× 重み付け
- 元請完成工事高
- 最大値: 約2,300点

**W: 社会性等（加点項目）**
- 防災協定締結: +15点
- 建設キャリアアップシステム（CCUS）: +20点
- 若年技術者育成: +6点
- ISO認証: +5点
- 範囲: 0〜1,966点

**P点の実例**

中堅ゼネコンの例（年間売上50億円、技術者20名）：
- X1: 1,200点
- X2: 800点
- Y: 900点
- Z: 1,100点
- W: 800点

→ P = 0.25×1,200 + 0.15×800 + 0.20×900 + 0.25×1,100 + 0.15×800
    = 300 + 120 + 180 + 275 + 120 = **995点**

#### 自治体別ランク基準の例

| 自治体 | Aランク | Bランク | Cランク | Dランク |
| --- | --- | --- | --- | --- |
| 東京都 | 1,000点以上 | 850〜999点 | 700〜849点 | 700点未満 |
| 神奈川県 | 950点以上 | 800〜949点 | 650〜799点 | 650点未満 |
| 地方中核市 | 800点以上 | 650〜799点 | 500〜649点 | 500点未満 |
| 小規模自治体 | 700点以上 | 550〜699点 | 400〜549点 | 400点未満 |

**AIの判別ロジック**

```python
def suggest_rank_eligibility(user_p_score: int, location: str, required_rank: str) -> str:
    rank_thresholds = get_rank_threshold(location)  # 自治体ごとの基準を取得

    if required_rank == "A" and user_p_score < rank_thresholds["A"]:
        return f"【警告】Aランク案件ですが、あなたのP点{user_p_score}は基準{rank_thresholds['A']}を下回っています"
    elif required_rank == "C" and user_p_score >= rank_thresholds["B"]:
        return f"【注意】Cランク案件ですが、あなたのP点{user_p_score}はランク過多の可能性があります"
    else:
        return "ランク要件を満たしていると思われます"
```

#### 地域要件の詳細

**地域区分の定義**

| 区分 | 定義 | 加点・制限 |
| --- | --- | --- |
| **市内業者** | 本店が市内にある | 最優先、小規模案件は市内限定 |
| **準市内業者** | 支店・営業所が市内にある | 市内業者の次点、中規模案件で認められる |
| **県内業者** | 本店または支店が県内にある | 大規模案件や専門工事で対象 |
| **県外可** | 制限なし | WTO案件や特殊技術が必要な場合 |

**地域要件の戦略的分析**

AIは以下のロジックで地域要件を解釈すべきです：

```
IF 案件規模 < 2,000万円 AND 工種 IN ["舗装", "道路改良", "維持補修"] THEN
  → 「市内業者限定の可能性が極めて高い」
  → ユーザーが市外企業なら「参加困難」と警告

ELSE IF 案件規模 >= 1億円 AND 工種 IN ["橋梁", "トンネル", "PC構造物"] THEN
  → 「県内または県外も可の可能性」
  → 「地域要件よりも技術力と実績が重視される」とアドバイス

ELSE IF "WTO" IN 案件名 THEN
  → 「地域制限なし（国際競争）」
```

**支店設置の実務**

多くの企業は、継続的に受注したい自治体に「支店」または「営業所」を設置します。ただし、以下の要件があります：

1. **常駐職員**: 最低1名の常駐が必要（名義貸しは違法）
2. **独立した事務所**: 実体のある事務所（バーチャルオフィスは不可）
3. **事前の届出**: 建設業許可の変更届が必要（数週間〜数ヶ月）

AIは「準市内業者」が参加可能な案件を見つけた場合、「支店設置を検討する価値がある地域か」という戦略的示唆も提供できます。

#### 配置技術者の要件

**主な資格**

| 資格 | 対象工種 | 配置義務 |
| --- | --- | --- |
| 1級土木施工管理技士 | 土木一式工事 | 3,500万円以上（元請） |
| 2級土木施工管理技士 | 土木一式工事 | 500万円〜3,500万円（元請） |
| 1級建築施工管理技士 | 建築一式工事 | 7,000万円以上（元請） |
| 技術士（建設部門） | 全工種 | 高度技術案件で優遇 |

**CORINS（工事実績データベース）**

CORINSは「Construction Records Information Service」の略で、過去の公共工事実績を登録・公開するシステムです。

AIが知るべきCORINSの重要性：
1. **総合評価での加点**: 同種工事の実績がある技術者の配置で大幅加点
2. **資格要件の証明**: 「過去5年以内に同規模の橋梁工事実績」などの要件確認
3. **競合分析**: 他社の実績を調査し、競合優位性を判断

**AIのアドバイス例**

```
「この案件は『過去5年以内に同種工事（PC橋上部工）の実績を有する技術者の配置』
が要件となっています。CORINSで貴社の登録実績を確認し、該当する技術者が
いない場合は、協力会社からの技術者派遣を検討してください。」
```
